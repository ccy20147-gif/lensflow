# Foundation 质量评测基线 — 测试套件

> Governed by: TF-QLT-001 | Version: 1.0 | Status: frozen
> Last updated: 2026-07-12

---

## 概述

本文件定义 LensFlow Foundation 层的六类质量评测套件。每类套件包含固定样本、适用范围、critical failure 规则和自动指标门槛。

六类套件覆盖：

1. **文本（Text）** — AI 生成文本的质量与一致性
2. **身份（Identity）** — 角色/人物身份一致性
3. **镜头（Camera）** — 镜头控制指令遵从性
4. **51 镜头（51 Shots）** — 多镜头叙事连贯性
5. **广告图（Ad）** — 广告/营销视觉质量
6. **交互（Interaction）** — UI/UX 交互质量

---

## 1. 文本套件（Text）

### 适用范围
AI Agent 生成的剧本、对白、旁白、Prompt 描述文本。

### 样本集

| ID | 输入 | 期望输出特征 | 来源 |
|---|---|---|---|
| TXT-001 | "写一段 30 字以内的产品简介：一个 AI 动画创作平台" | 中文通顺、产品定位准确、≤30 字 | 人工撰写 |
| TXT-002 | "用 50 字描述一个雨后城市夜景的动画场景" | 包含场景、光线、氛围描述 | 人工撰写 |
| TXT-003 | "Translate to EN: 这个角色有一双蓝色的眼睛和红色的头发" | 英文准确、不丢失属性 | 人工撰写 |
| TXT-004 | "镜头：俯拍，从广场中心缓慢上升" | 正确解析镜头指令 | 人工撰写 |

### Critical Failures
- 输出为空或仅返回错误代码
- 关键属性丢失（如角色名字、颜色、位置）
- schema 字段缺失或类型错误

### 自动指标
- Schema 遵从率：100%
- 非空率：100%
- 字段完整率：≥95%

## 2. 身份套件（Identity）

### 适用范围
角色/人物视觉身份一致性 — 同一角色在不同场景、镜头、服饰下的可识别性。

### 样本集

| ID | 输入 | 期望输出特征 | 来源 |
|---|---|---|---|
| IDN-001 | 角色 A 正面肖像 + "稍侧脸微笑" | 面部特征一致（眼型、发型、肤色） | 人工标注 |
| IDN-002 | 角色 A + "穿红色外套站在雨中" | 角色面部不变，服饰改变合理 | 人工标注 |
| IDN-003 | 角色 B 全身像 + "坐在咖啡馆" | 全身像身份可关联到面部特征 | 人工标注 |
| IDN-004 | 角色 A + 角色 B + "握手" | 两个角色各自的身份特征都保持 | 人工标注 |

### Critical Failures
- 面部特征明显改变（眼型、鼻型、肤色整体漂移）
- 两个角色特征互换或融合
- 输出完全不可识别为人形

### 自动指标
- 特征向量余弦相似度（vs 参考角色 embedding）：≥0.70
- 生成成功率：100%

### 人工 Rubric（盲评）
| 分档 | 标准 |
|---|---|
| 5 - 完美 | 身份完全可辨认，与参考图一致 |
| 4 - 良好 | 身份可辨认，有轻微变化但不影响识别 |
| 3 - 及格 | 身份基本可辨认，但存在明显差异 |
| 2 - 不足 | 身份部分丢失，难以确认是否为同一角色 |
| 1 - 失败 | 身份完全改变或不可辨认 |

## 3. 镜头套件（Camera）

### 适用范围
镜头控制指令的视觉遵从性 — 相机角度、运动、景别、焦距。

### 样本集

| ID | 输入 | 期望输出特征 | 来源 |
|---|---|---|---|
| CAM-001 | 场景 + "特写：角色眼睛" | 画面聚焦眼睛区域 | 人工撰写 |
| CAM-002 | 场景 + "俯拍：鸟瞰城市" | 透视关系符合俯拍 | 人工撰写 |
| CAM-003 | 场景 + "推镜头：从全景推进到面部" | 构图变化符合推镜头效果 | 人工撰写 |
| CAM-004 | 场景 + "仰拍：角色仰望高楼" | 透视关系符合仰拍 | 人工撰写 |
| CAM-005 | 场景 + "第一人称视角：穿过门廊" | 模拟第一人称视野 | 人工撰写 |

### Critical Failures
- 镜头类型与实际生成结果不符（如要求俯拍得到平拍）
- 多人场景中焦段/景深与描述不符
- 运动方向与描述相反

### 自动指标
- 镜头类型匹配度：≥80%（通过视觉分类模型验证）
- 构图规则遵从率：≥90%

### 人工 Rubric（盲评）
| 分档 | 标准 |
|---|---|
| 5 - 完美 | 镜头指令完全遵从，构图专业 |
| 4 - 良好 | 镜头指令基本遵从，有细微偏差 |
| 3 - 及格 | 镜头指令部分体现在画面中 |
| 2 - 不足 | 镜头指令大部分未体现 |
| 1 - 失败 | 镜头指令完全未被遵循 |

## 4. 51 镜头套件（51 Shots）

### 适用范围
多镜头序列的叙事连贯性 — 角色、场景、光照、视角在多个镜头间的一致性。

### 样本集

| ID | 输入 | 期望输出特征 | 来源 |
|---|---|---|---|
| SHT-001 | 3 镜头故事板："①全景清晨教室 ②中景学生看书 ③特写翻页" | 角色一致、光照连续、构图递进 | 人工绘制 |
| SHT-002 | 5 镜头追逐场景 | 同一场景不同角度的空间一致性 | 人工撰写 |
| SHT-003 | 同一角色 4 种不同情感状态 | 角色身份一致，表情情感正确 | 人工标注 |
| SHT-004 | 日景→夜景过渡 3 镜头 | 光照色温过渡自然 | 人工撰写 |

### Critical Failures
- 空间布局在镜头间不一致（门的位置改变、物体消失/出现）
- 同一角色在不同镜头中面部特征不一致
- 光照条件在连续镜头中跳跃式变化而无合理过渡

### 自动指标
- 角色 embedding 跨镜头一致性：≥0.65
- 场景布局关键点匹配率：≥70%
- 光照参数方差（相邻镜头）：≤0.3（归一化）

### 人工 Rubric（盲评）
| 分档 | 标准 |
|---|---|
| 5 - 完美 | 所有镜头角色、场景、光照完全一致 |
| 4 - 良好 | 整体一致，存在可接受的细微差异 |
| 3 - 及格 | 核心元素一致，次要元素有变化 |
| 2 - 不足 | 角色或场景出现不应有的变化 |
| 1 - 失败 | 关键元素在镜头间不一致，破坏叙事 |

## 全局规则

### Critical Failure 优先级
任何 critical failure 自动阻断 Quality Gate，不受平均分影响。

### 双人盲评规则
1. 每份主观样本至少由两名评审员独立评分
2. 分差 ≤1 档：取平均值（向下取整）
3. 分差 ≥2 档：自动触发第三人裁决
4. 评审员不可见对方评分和模型品牌

### 回归基线
| 指标 | Foundation 门槛 | 数据来源 |
|---|---|---|
| Schema 遵从率 | 100% | 自动校验 |
| Critical failure 数 | 0 | 自动 + 人工 |
| 人工评分均值 | ≥3.0/5.0 | 盲评 |
| 双评一致率 | ≥70% | 统计 |
| 生成成功率 | ≥95% | 自动 |
| 最大样本失败率 | ≤10% | 统计 |
