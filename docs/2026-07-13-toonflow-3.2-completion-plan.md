# LensFlow 3.2 Completion Plan

## Objective

Complete every 3.2 PRD (`TF-WF-001` through `TF-WF-010`, `TF-AGT-001`
through `TF-AGT-006`, and `TF-MR-001`) to its stated FR/AC, replacing
the current demo-only paths and explicit control-flow stubs.  A PRD is not
complete merely because CRUD, a static validator, or a screen exists.

## Provider Decision

All model traffic goes through AtlasCloud.  The backend owns the integration:

- `ATLASCLOUD_API_KEY` is an environment/credential binding only; it must
  never enter a browser bundle, database payload, audit event, or exception.
- The adapter supports the documented AtlasCloud LLM, image, video, prediction
  status, webhook and upload-file paths.  It maps each request to a frozen
  provider/model/capability snapshot and an idempotency key.
- Before any external request, persist `NodeRunAttempt`,
  `ProviderInvocationAttempt`, and `provider_dispatch` outbox event in one
  transaction.  Results/reconciliation persist bindings, actual usage/cost,
  trace and `result_publish` in one transaction.
- Real AtlasCloud tests are opt-in (`ATLASCLOUD_API_KEY` plus an explicit test
  flag).  Contract tests use an HTTP transport fake and prove payload,
  idempotency, timeout, retry and unknown-submission behavior without a key.

AtlasCloud references: `/docs/models/llm`, `/docs/models/image`,
`/docs/models/video`, `/docs/predictions`, `/docs/webhooks`,
`/docs/upload-files`, `/docs/createChatCompletion`, `/docs/getResult`.

### Explicit live smoke protocol

`backend/tests/test_atlascloud_live_smoke.py` is intentionally excluded by
default. It requires both `ATLASCLOUD_API_KEY` and
`ATLASCLOUD_LIVE_SMOKE_ENABLED=1`, then checks documented chat, upload, image
submission plus prediction polling. Video additionally requires
`ATLASCLOUD_LIVE_SMOKE_VIDEO_ENABLED=1`, because submitting even a minimal
480p/5-second video can incur a non-trivial charge. The model and JSON payload
for each class are overridable through `ATLASCLOUD_LIVE_SMOKE_{CHAT,IMAGE,VIDEO}_MODEL`
and `..._PAYLOAD_JSON`; request timeout is bounded to 1-60 seconds through
`ATLASCLOUD_LIVE_SMOKE_TIMEOUT_SECONDS`. The suite never prints credentials,
response bodies, prediction IDs, or temporary media URLs.

For the async callback acceptance, set `ATLASCLOUD_WEBHOOK_URL` to the public
HTTPS URL ending in `/api/v1/runtime/callbacks/atlascloud`. AtlasCloud signs
terminal media callbacks with Ed25519 headers and publishes public keys from
`/api/v1/webhooks/jwks.json`; the backend verifies the documented timestamp,
key ID, session ID, and signature before it accepts a result. Do not enable
`ATLASCLOUD_LEGACY_WEBHOOK_HMAC_ENABLED` for a new deployment. That flag exists
only for a time-bounded provider migration and accepts the documented legacy
`X-AtlasCloud-Webhook-Signature` header, never the former local header.

## Mandatory Acceptance Gates

1. Each PRD has tests that exercise every declared AC, including negative,
   authorization, concurrency and restart cases where specified.
2. No `stub`, `demo_note`, or "not enabled" code path may stand in for an
   in-scope V1 requirement.
3. All state-changing runtime behavior is PostgreSQL-backed and transactionally
   paired with appropriate outbox events.
4. API authentication and owner-scope authorization are enforced; setup-only
   helpers are not production endpoints and cannot bypass workflow ownership.
5. Required quality gates pass: requirements validator, Alembic upgrade and
   roundtrip, PG test suite, ruff, mypy, frontend typecheck/lint/build, and
   browser E2E against a clean backend/frontend/PostgreSQL start.
6. PRD metadata and tracker advance only after independent review confirms the
   preceding evidence.  Provider live smoke tests remain explicitly skipped
   when no authorized key is supplied; their absence cannot be represented as
   successful generation.
